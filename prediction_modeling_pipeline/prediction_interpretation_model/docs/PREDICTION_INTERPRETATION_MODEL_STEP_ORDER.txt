PREDICTION INTERPRETATION MODEL STEP ORDER

00_run_prediction_interpretation_model.py
Orchestrator. Runs implemented steps in order. Current default is Step 01 only.

01_prepare_interpretation_inputs.py
Validate the V2 full run and collect source tables into stable prepared inputs.

02_build_feature_and_treatment_dictionary.py
Create readable feature dictionary, biology theme dictionary, treatment dictionary, and treatment component table.

03_compute_signed_spatial_effects.py
Compute signed feature and theme effects for sensitivity-associated versus resistance-associated interpretation.

04_build_treatment_interpretation_cards.py
Create one final interpretation card per validated treatment.

05_build_sample_level_interpretations.py
Create sample-level spatial sensitivity/resistance interpretation tables.

06_build_mechanism_atlas.py
Build cross-treatment mechanism and biology theme matrices.

07_make_final_outputs.py
Generate final figures, captions, final tables, Excel workbook, final report, and slide narrative.

08_qc_and_package_final_outputs.py
Run final QC and create final ZIP/package.

Policy:
Use the spatial_prediction_model_V2 full run as source truth.
Do not rerun V2.
Do not perform open model selection.
Do not use deprecated outputs as final source truth.
Do not make clinical treatment recommendations.
Do not treat V2 SHAP/gain as causal biology.
